# /budget-architect

调用 `budget-architect` skill，进入企业预算编制与预算管理模式。

## 何时使用

当你需要处理"未来的钱怎么花、目标怎么切、执行怎么盯"这类前瞻性财务规划问题时：

- **年度预算编制**：Q3-Q4 编制来年预算，需要方法选择、目标下达、部门汇总、对账收敛
- **零基预算（ZBB）**：需要一次成本大清洗、砍掉惯性开支、重新论证每一分钱
- **滚动预测**：想把僵化的年度预算改成 12+3 / 18 个月滚动窗口，跟上业务节奏
- **差异分析**：月度/季度预算 vs 实际差异拆解（价差、量差、混合差异），找根因
- **预算预警机制**：建立预警阈值和触发动作，让预算超支不再"事后才知道"
- **部门预算协同**：销售/生产/研发/HR/IT 各部门预算的口径统一、勾稽和冲突调解
- **FP&A 体系搭建**：从零建立财务规划与分析（Financial Planning & Analysis）流程

## 与 finance-manager 的边界

- **finance-manager 看历史报表**：三大报表分析、财务比率、杜邦、DCF 估值、现金流预警——回答"这家公司过去/现在财务上到底怎么样"
- **budget-architect 看未来预测**：预算编制、滚动预测、差异分析、目标下达——回答"下一年/下一季度我们钱怎么规划、怎么盯执行"

两个 skill 会有交叉（比如预算 DCF 假设来自历史财报），但分工清晰：**过去-现在归 finance-manager，未来归 budget-architect**。

## 输入建议

尽量一次提供以下要素：

1. **公司基本信息**：行业、规模（营收/员工数）、发展阶段（快速增长/成熟/衰退）、股权性质（民企/外资/央国企）
2. **预算目的**：年度编制 / 季度调整 / 滚动更新 / 差异复盘 / 部门协调
3. **历史基线**：至少去年实际数、今年预算数、今年 YTD 实际数（用于判断趋势）
4. **战略输入**：董事会/股东下达的目标（营收、利润、市占率、投资），或最少给个"方向"
5. **组织结构**：涉及哪些部门/BU，是否有海外/子公司需要合并
6. **预算文化**：偏自上而下（目标刚性）还是自下而上（部门自主），有没有历史遗留的预算游戏问题

信息不全也可以先开始，`budget-architect` 会显式标注假设并给出置信度。

## 默认输出

- **预算编制方案**：方法选择（增量 / 零基 / 滚动 / ABB）、假设、编制逻辑与时间线
- **自上而下 / 自下而上对账表**：总部目标 vs 部门汇总，差异与调整方案
- **滚动预测表（12 个月）**：分月的收入、成本、毛利、期间费用、经营利润、现金流关键行
- **差异预警机制**：预警阈值（红/黄/绿）、触发动作、复盘节奏
- **数据假设与依赖标注**：明确哪些是用户给的、哪些是合理假设、缺哪些数据
