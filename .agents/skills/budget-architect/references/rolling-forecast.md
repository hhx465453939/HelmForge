# 滚动预测

**滚动预测（Rolling Forecast）不是取代年度预算，而是给年度预算装上"雷达"**：年度预算给董事会承诺，滚动预测给管理层运营。两者并行，各司其职。

## 1. 为什么需要滚动预测

### 1.1 传统年度预算的三大痛点

- **僵化**：1 月做完的预算，2 月宏观突变（如疫情、加息、大客户流失）就过时了
- **周期错位**：预算按日历年，业务按行业周期（如零售看双 11/春节，SaaS 看合同月）
- **决策滞后**：等到年中发现预算不对，已经错过 6 个月机会窗口

### 1.2 滚动预测的核心价值

- **前瞻性一致**：任何时点都有未来 12 个月的完整预测
- **决策及时**：每季度更新一次，最多滞后 3 个月
- **打破日历约束**：预测不再受限于财年边界

## 2. 窗口设计

### 2.1 常见窗口类型

| 窗口 | 结构 | 适用场景 |
|---|---|---|
| 12+3 | 每季度更新未来 12 个月，向前推 3 个月 | 最经典、适用面最广 |
| 18 月固定 | 永远看未来 18 个月 | 长周期业务（Capex 密集、地产） |
| 8+4 | 未来 8 个月月度 + 后 4 个月季度 | 精度前重后轻，节省更新成本 |
| N+M（灵活） | N 月月度 + M 月季度 | 定制化，通常用于合并子公司 |
| 24 月固定 | 永远看未来 24 个月 | 战略型预测，颗粒度粗 |

### 2.2 12+3 窗口详解（推荐默认）

```
时点：Q1 结束
      当前月 →                                                       预测终点 →
      Apr    May    Jun    Jul    Aug    Sep    Oct    Nov    Dec    Jan    Feb    Mar    Apr    May    Jun
      [────────────────── 12 个月预测 ──────────────────][── 前瞻 3 个月 ──]

时点：Q2 结束（新的滚动）
      当前月 →                                                       预测终点 →
      Jul    Aug    Sep    Oct    Nov    Dec    Jan    Feb    Mar    Apr    May    Jun    Jul    Aug    Sep
      [────────────────── 12 个月预测 ──────────────────][── 前瞻 3 个月 ──]
```

每季度往前"抽掉"3 个月过去（已成 Actual），"添加"3 个月未来。

### 2.3 更新颗粒度

- **月度预测**：未来 6-12 个月按月拆分
- **季度预测**：更远的 6-12 个月按季度即可
- **日/周预测**：仅用于运营层（库存、现金），不进滚动预测框架

## 3. 驱动因子拆解

**滚动预测的核心不是数字，是驱动因子模型**。每个数字背后要有可更新的假设。

### 3.1 收入驱动因子

**通用公式**：`收入 = 客户数 × 平均订单 × 频次`

按业务类型细化：

- **零售/电商**：`收入 = 流量 × 转化率 × 客单价`
- **SaaS**：`收入 = 期初 ARR + 新增 ARR - 流失 ARR + 拓展 ARR`
- **制造业**：`收入 = 产能 × 利用率 × 平均单价`
- **服务业**：`收入 = 顾问人数 × 计费小时 × 小时单价`
- **广告业**：`收入 = 曝光量 × CPM 或 转化数 × CPC`

### 3.2 成本驱动因子

- **可变成本**：随销量线性变化（原材料、包装、运费）→ `= 销量 × 单位成本`
- **半可变成本**：阶梯式（设备折旧、租金）→ 按销量分档
- **固定成本**：与销量无关（管理层薪资、办公租金）→ 独立预测

### 3.3 费用驱动因子

- **销售费用**：`= 营销投入 + 销售提成` → 营销投入用 ROI 模型，销售提成 = 收入 × 提成率
- **管理费用**：`= 编制 × 人均成本` → 编制随业务分档
- **研发费用**：`= 项目数 × 项目成本` → 按项目 pipeline 预测

### 3.4 现金流驱动因子

滚动预测必须包含现金流预测（很多公司只做 P&L 滚动，是错的）：

- **经营性现金流** = 净利润 + 折旧摊销 - 应收账款增加 - 存货增加 + 应付账款增加
- **投资性现金流** = -Capex（按项目立项时点分月）
- **筹资性现金流** = 融资 - 还款 - 分红

## 4. 更新节奏与流程

### 4.1 标准节奏（季度更新）

- **Wk1**：Actual 数据锁定（月末结算完成）
- **Wk2**：业务部门更新驱动因子假设（sales pipeline、生产计划、招聘计划）
- **Wk3**：财务合并计算，识别与上季度预测的重大差异
- **Wk4**：CFO 评审 + 高管沟通会 + 董事会/股东同步

### 4.2 加速节奏（月度更新）

适用于高不确定性业务（初创、大宗商品交易）。

- 每月 5 号前 Actual 锁定
- 每月 10 号前更新完成
- 15 号进管理层运营会

### 4.3 触发式更新（应急）

以下事件触发即时重做（不等季度）：

- 单一大客户丢失/新签（>10% 营收）
- 关键原材料价格波动 >20%
- 汇率剧烈波动（>5%）
- 监管政策重大变化
- 疫情/自然灾害/供应链断裂
- 收购/剥离完成

## 5. 与年度预算的关系

### 5.1 两者定位

| 维度 | 年度预算 | 滚动预测 |
|---|---|---|
| 用途 | 董事会/股东承诺、KPI 挂钩 | 管理层运营参考 |
| 更新频率 | 一年一次 | 每季度/每月 |
| 精度 | 前 3 月细，后 9 月粗 | 全 12 月保持相似精度 |
| 心理契约 | 是承诺 | 是判断 |
| 与实际差异 | 通常 ±15-20% | 期望 ±5-10% |

### 5.2 并行运行时的原则

- **KPI 挂钩用年度预算**：否则每次滚动改一次，员工无所适从
- **运营决策用滚动预测**：更贴近现实，指导资源调配
- **重大偏离时同步年度**：如滚动预测显示营收比年度预算低 20%，需正式重议年度目标（Reforecast）
- **不重复劳动**：滚动预测的假设变化，可以自动更新年度剩余月份

### 5.3 Reforecast（正式重预测）

滚动预测 vs 年度预算差异过大时，需要正式 Reforecast，走审批流程：

- 差异 <10%：滚动预测内部消化，年度预算不动
- 差异 10-20%：管理层内部 Reforecast，董事会知情
- 差异 >20%：正式 Reforecast，董事会审批新的年度目标

## 6. 疫情/黑天鹅期的应急滚动

### 6.1 应急切换动作

当业务遭遇突发大变故（疫情封控、供应链断裂、政策突变）：

- **周期缩短**：从季度更新改为月度、甚至双周
- **窗口缩短**：从 12+3 改为 6+3 或 3+3（长期看不清）
- **情景加多**：单一预测变三情景（乐观/中性/悲观），并给每个情景概率
- **驱动因子降维**：只保留 top 5-10 关键因子，其他简化为占比
- **现金流优先**：把 P&L 后置，先保 13 周现金流预测（防止现金链断裂）

### 6.2 应急预测模板（13 周现金流）

```
| 周次 | 期初现金 | 收入(入账) | 应收回款 | 融资流入 | 现金总流入 | 工资 | 供应商 | 租金水电 | 税费 | 利息 | 现金总流出 | 期末现金 | 现金充足天数 |
| W1   | ...      | ...        | ...      | ...      | ...        | ... | ...    | ...      | ...  | ...  | ...        | ...      | 45 |
| W2   | ...      | ...        | ...      | ...      | ...        | ... | ...    | ...      | ...  | ...  | ...        | ...      | 42 |
| ...  |          |            |          |          |            |     |        |          |      |      |            |          |    |
| W13  | ...      | ...        | ...      | ...      | ...        | ... | ...    | ...      | ...  | ...  | ...        | ...      | 20 |
```

如果现金充足天数持续下降到 <30 天，进入危机预案。

## 7. 快速增长期 vs 成熟期的滚动重点

### 7.1 快速增长期（营收 YoY > 30%）

- **重点**：收入驱动因子的可靠性（客户 pipeline、CAC、LTV）
- **风险**：过度乐观，把最好月份外推
- **窗口**：偏短（8+4 或 6+3），因为业务变化快
- **KPI 挂钩**：新增客户数、ARR 增长、DAU/MAU

### 7.2 成熟期（营收 YoY < 15%）

- **重点**：成本控制、毛利率维护、现金分红能力
- **风险**：忽视新兴威胁（技术替代、竞对创新）
- **窗口**：可用 12+3 或 18 月固定
- **KPI 挂钩**：EBIT 增速、经营性现金流、ROE

### 7.3 衰退期（营收 YoY < 0）

- **重点**：现金保全、资产处置
- **风险**：预测过度乐观，掩盖真实衰退速度
- **窗口**：短且高频（月度 6+3）
- **KPI 挂钩**：现金流为正、库存周转、员工留存

## 8. 常见坑

1. **只做 P&L 不做现金流**：预测利润上升，但现金流断裂（大客户不付款）→ 必须包含现金流预测
2. **不更新驱动因子只改数字**：把上季度数字整体 +5% 交差 → 强制要求写变化说明
3. **假设无人挑战**：CFO 不看每个假设的合理性 → 每季度 CFO 挑战会必须有
4. **滚动完全替代年度**：董事会没有承诺目标，管理层不知道要冲什么 → 必须并行
5. **系统跟不上**：Excel 支撑不了大公司多部门滚动 → 需要 Anaplan / Adaptive / Vena 等
6. **组织不接受**：中层觉得"我年初已经承诺过了，你现在又要我改" → CEO 必须明确"滚动是判断不是承诺"
7. **驱动因子失效**：疫情期继续用"过去 12 个月客户数外推" → 必须换驱动因子（如"疫情后 3 个月月环比"）

## 9. 滚动预测成熟度阶梯

- **Level 1**：季度更新 12 月 P&L 预测（Excel）—— 60% 中小企业停在这里
- **Level 2**：加入现金流预测、情景分析、驱动因子模型
- **Level 3**：月度更新 + 应急触发机制 + 分部门 P&L
- **Level 4**：专业系统（Anaplan / Adaptive）+ 与 ERP 数据直连
- **Level 5**：机器学习驱动的假设更新（如自动根据宏观数据调整假设）—— 顶级公司

大部分中国民企需要从 Level 1 到 Level 2，就是巨大提升。
