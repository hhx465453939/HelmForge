# Talk Track And Message Templates

默认先给短版本，再给完整版。

## 1. 试探需求

```text
我先和您对齐一下，这轮您最想优先解决的是预算、时间，还是执行方式？
```

## 2. 守价但不硬顶

```text
这个价格不是随便报出来的，核心是对应现在这套投入和交付强度。
如果您这边预算确实有框，我们可以一起调整范围或节奏，但不太建议简单砍总价。
```

## 3. 给台阶的让步

```text
如果您这边能把付款节点往前收一点，我这边可以在这个部分帮您留一点空间。
```

## 4. 催回款

```text
我先和您确认一下，这笔款现在卡住的具体环节是流程、预算，还是内部签批？
如果方便的话，给我一个最晚反馈时间，我这边好同步安排后续。
```

## 5. 推进合作

```text
我理解这事您还在内部评估。为了避免我们双方都空转，我建议这轮先只确认一件事：是否继续往下推进。
如果继续，我这边就按这个版本准备下一步；如果暂时不推进，也没关系，我们把窗口留住。
```

## 6. 对方压价

```text
如果只是把总价往下压，最后通常会落到执行质量和响应速度上。
更现实的做法是您告诉我最紧的限制在哪，我来帮您重排结构。
```

## 7. 逼单收口

```text
现在核心分歧其实只剩这两个点。只要这两个点对齐，这事就可以往下走了。
您这边如果认可，我就按这个口径推进，不再来回改版本。
```

## Template Rule

每次输出尽量给：

- `短版`：适合微信/飞书
- `稳版`：适合邮件/正式沟通
- `强版`：适合收口、施压、明确边界
