# Local Fit

这个 skill 要贴合你本地已经验证过的谈判体验，而不是写成抽象纵横学。

## What Worked Locally

从 `memory/2026-03-19-cooperation-negotiation.md` 可以提炼出几条非常实用的本地规则：

### 1. 先看谁更急

- 如果对方已经主动追进度、追预算、追细节，说明他不是“随便看看”
- 一旦确认对方真实有需求，策略要从纯观望转成“控制推进”

### 2. 颗粒度要匹配阶段

- 初步沟通不要一次把所有合同、条款、付款、专票、风险都摊死
- 但也不能太空，至少要让对方能往下内部流转

### 3. 少给决策点

- 一条消息里不要让对方同时决定太多问题
- 优先让对方回答最关键的那个问题

### 4. 让步要留口子

- 例如“如果专票协调不了，就按普票准备，您看可以吗？”
- 这类表达的价值在于：提前打预防针，但不把自己锁死

### 5. 催得太紧会丢主动权

- 对方还没表现真实需求前，别自己先冲
- 但对方已经明确在推进时，也别继续装冷导致错过窗口

## UX Failure To Avoid

### P0

- 只讲原则，不给可直接发的话术
- 不判断阶段，就直接写长篇模板

### P1

- 一条消息里给 4-5 个选择题
- 还没拿到承诺就过度让步
- 为了显得专业而写得像合同，不像真实沟通

### P2

- 只给一个版本，不给“对方怎么回、我下一句怎么接”

## Local Trigger Phrases

这个 skill 应该在这些说法里自然触发：

- `帮我跟他谈`
- `这句话怎么说`
- `这个怎么回`
- `帮我压一下价`
- `怎么催回款`
- `怎么推进合作`
- `给我一个能谈成的版本`
- `对方下一句可能会怎么回`
- `还有什么我没想到`
