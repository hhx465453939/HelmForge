# Problem Taxonomy

先分类，再分析。避免把完全不同的问题混成一个“大法律问题”。

## A. 争议与抗辩

适用：

- 起诉
- 应诉
- 仲裁
- 执行
- 保全
- 和解

核心任务：

- 设计我方主张
- 预判对方抗辩
- 找破局点
- 建证据矩阵

## B. 调查与追偿

适用：

- 电诈追偿
- 账户冻结
- 不当得利
- 资金流向
- 地下钱庄
- 民刑交叉
- 内部舞弊

核心任务：

- 还原资金与行为路径
- 找明知/参与/受益/异常特征
- 预判包装性抗辩
- 输出追偿与破局策略

## C. 合规与监管

适用：

- 监管问询
- 处罚
- 反洗钱
- 数据合规
- 医疗/金融/广告/平台合规
- 内控整改

核心任务：

- 判断违规风险级别
- 设计整改顺序
- 形成对外口径
- 避免次生风险

## D. 合同与交易

适用：

- 合同审查
- 谈判
- 交易结构
- 并购 / 投融资 / 商业合作

核心任务：

- 找关键条款
- 识别不对称风险
- 设计杠杆、条件、退出和救济

## E. 公司治理与专项法务

适用：

- 股权
- 董事会
- 劳动
- 商业秘密
- 知识产权
- 跨境与数据

核心任务：

- 判断控制权与责任边界
- 找最容易爆雷的点
- 输出治理或专项处理方案

## Routing Rule

一个问题可能跨多个域，但要先定主域：

- 若结果主要取决于法院/仲裁/抗辩 -> 主域选 A
- 若结果主要取决于资金、线索、证据链还原 -> 主域选 B
- 若结果主要取决于监管和整改 -> 主域选 C
- 若结果主要取决于条款设计和签约顺序 -> 主域选 D
- 若结果主要取决于组织权利义务边界 -> 主域选 E
