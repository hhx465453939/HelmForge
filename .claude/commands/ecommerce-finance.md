# /ecommerce-finance - 电商财务费用处理

根据用户提供的支付宝账务明细CSV文件，执行完整的电商财务费用处理流程。

## 执行步骤

### 1. 确认输入文件

确认用户工作目录中的CSV文件：
- 找到 `*账务明细_*.csv` (明细文件) 和 `*账务明细(汇总)_*.csv` (汇总文件)
- 如果用户指定了路径则使用指定路径
- 从文件名中提取账号ID、账期信息

### 2. 执行处理脚本

```bash
python <skill_dir>/scripts/process_alipay.py \
  --input "<明细CSV>" \
  --summary "<汇总CSV>" \
  --output "<店铺名>支付宝明细<账期>.xlsx" \
  --store "<自动推断或用户指定>" \
  --period "<从文件名提取>"
```

### 3. 结果汇报

处理完成后向用户报告：
- 记录总数与分类覆盖率
- 订单数量与费用合计
- 闭环校验结果 (8项是否全部通过)
- 如有未分类记录，列出并请求确认

### 4. 异常处理

如果闭环校验不通过：
- 明确指出哪项失败、差异金额是多少
- 建议排查方向（数据缺行？新增费用类型？跨期错配？）
- 不要在未确认的情况下强行修改分类规则

## 参数

`$ARGUMENTS` 可以是：
- 空 -> 自动在当前目录搜索CSV文件
- 文件路径 -> 指定要处理的CSV
- `check` -> 仅执行闭环校验不生成报表
- `extend` -> 进入功能扩展模式（会计老师添加新需求）

## 功能扩展模式 (/ecommerce-finance extend)

当用户说 `/ecommerce-finance extend` 时，引导用户按以下模板描述需求：

```
请告诉我：
1. 【功能名称】这个新功能叫什么？
2. 【输入】需要什么数据？
3. 【输出】期望看到什么结果？
4. 【规则】核心计算逻辑或分类规则是什么？
5. 【示例】给一两条样例数据
```

收集完信息后：
1. 在 `scripts/` 下生成新模块
2. 修改 `process_alipay.py` 集成新功能
3. 更新 `references/mapping_schema.json`
4. 运行测试验证闭环
