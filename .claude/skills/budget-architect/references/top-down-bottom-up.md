# 自上而下 vs 自下而上

预算编制最难的不是算数字，是让"CEO 想要的目标"和"部门总监能接的承诺"匹配起来。这是一场谈判，不是一场计算。

## 1. 两种方法的本质

### 1.1 自上而下（Top-Down）

**流程**：董事会/CEO 定总目标 → 拆解到 BU → 拆解到部门 → 拆解到个人 KPI

**逻辑起点**：股东回报、资本市场承诺、战略野心

**典型语言**：
- "明年营收增长必须 30%"
- "毛利率不能低于 40%"
- "费用率必须降到 20% 以内"

### 1.2 自下而上（Bottom-Up）

**流程**：一线业务员/项目经理估算 → 部门汇总 → BU 汇总 → 公司总账

**逻辑起点**：客户订单、产能、员工工时、真实市场感知

**典型语言**：
- "华东区明年能接 3.5 亿订单"（销售）
- "现有产线满负荷是 2 亿件，扩产需要 6 个月"（生产）
- "研发部明年需要新增 15 个工程师"（研发）

### 1.3 差异的本质

| 维度 | 自上而下 | 自下而上 |
|---|---|---|
| 出发点 | 应然（should） | 实然（is） |
| 天然倾向 | 激进、乐观 | 保守、防御 |
| 主要风险 | 目标脱离实际 | 目标缺乏野心 |
| 谁受益 | 股东 / 资本市场 | 部门 / 一线员工 |
| 谈判筹码 | 战略、资本、股价 | 客户、产能、员工 |

**关键洞察**：自上而下和自下而上不是二选一，而是必须都做，然后对账、谈判、收敛。没做自下而上的自上而下是拍脑袋；没做自上而下的自下而上是躺平。

## 2. 完整流程：三轮对账模型

### 2.1 第一轮：目标下达 + 部门初稿（Wk1-2）

- CEO/CFO 下达"框架性目标"：营收 +25%、毛利率 38%、期间费用率 22%
- 各部门独立编制初稿（不看总部数字）
- 财务汇总部门数字，形成"部门汇总总数"
- 计算差异：目标 vs 部门汇总

**典型第一轮差异**：
- 营收差异 -15% ~ -25%（部门保守，沙袋 10-20%）
- 毛利率差异 -2 ~ -5pp（部门低估提价能力）
- 期间费用差异 +5% ~ +15%（部门抢编制、抢费用）

### 2.2 第二轮：分歧讨论 + 谈判（Wk3-4）

**方法一：一对一谈判（民企/中小公司常见）**

- CFO 挨个约谈部门总监
- 逐科目讨论差异
- 用历史数据、行业基准挑战部门假设

**方法二：预算听证会（外资/大型企业常见）**

- 每个部门 1 小时汇报
- CEO/CFO/COO 三方联合挑战
- 现场初步定调，会后 3 天内补数据

**关键谈判筹码**：
- CEO 手上：股东承诺、战略窗口、行业机会
- 部门手上：客户名单、产能限制、员工现状

**收敛策略**：
- 硬约束：营收增速、毛利率、EBIT（董事会已承诺，不可动）
- 弹性变量：新增编制、营销费、Capex（可以谈）
- 交换筹码：给你多 5 个编制，营收目标必须 +2 亿

### 2.3 第三轮：终稿 + 承诺挂钩（Wk5-6）

- 收敛后的数字进入 KPI 挂钩
- 部门总监签"预算承诺函"（这一步很关键，把口头承诺变成书面）
- CFO 编制敏感性分析（乐观 / 中性 / 悲观三情景）
- 提交董事会审批

## 3. 分歧处理框架

### 3.1 差异分类

按差异大小和处理方式分类：

| 差异幅度 | 处理方式 | 决策层级 |
|---|---|---|
| <5% | 部门自行调整 | 部门总监 |
| 5-15% | CFO+部门总监一对一谈判 | CFO |
| 15-30% | 预算听证会+CEO 介入 | CEO |
| >30% | 战略重议，可能需要重新审视目标本身 | 董事会 |

### 3.2 根因诊断：为什么差异这么大？

自下而上偏低的原因（部门保守）：

- **沙袋（Sandbagging）**：故意报低，留出超额完成的空间（获得奖金）
- **风险规避**：对客户/市场不确定性过度悲观
- **信息不对称**：部门只看自己的一亩三分地，看不到宏观机会
- **激励错位**：完成 100% 拿 100% 奖金，超额 5% 只多 10% 奖金——不划算冲高

自上而下偏高的原因（总部激进）：

- **资本市场压力**：股价、股东承诺、上市对赌
- **战略野心**：CEO 想在任期内做出成绩
- **信息不对称**：总部只看财务数字，不知道客户在流失/员工在离职
- **参考错位**：拿最好季度外推，忽略淡旺季

### 3.3 干预动作对照表

| 诊断 | 干预动作 |
|---|---|
| 部门沙袋 | 改激励结构：超额 30% 拿 200% 奖金 |
| 部门风险规避 | 提供情景分析工具，让部门看到概率分布而不是单点 |
| 部门信息不对称 | 让 CEO/CMO 讲一次战略，同步宏观和竞对信息 |
| 部门激励错位 | 重设 KPI 结构：base + stretch + moonshot 三档 |
| 总部资本市场压力 | 承认这是硬约束，转而讨论"如何达到"而不是"能不能达到" |
| 总部战略野心 | 引入外部标杆（同行、投行报告），检验野心是否有基础 |
| 总部信息不对称 | 让 CEO 参加一次客户拜访，看真实市场 |

## 4. 混合模式：Counter-Flow / 松紧带

现实中很少是纯上或纯下，最常用的混合模式：

### 4.1 Counter-Flow（对撞式）

- 总部下达"松紧带"目标：营收 +20-30%、毛利率 35-40%
- 部门在松紧带内自主选择承诺点（选高的额外奖励，选低的减少弹性池）
- 中间碰撞，最终收敛

### 4.2 松紧带（Stretch Budget）

- Base 预算：部门承诺一定达到（挂 base 奖金）
- Stretch 预算：超额目标，达到给额外激励
- Moonshot：登月目标，达到给股权激励

优点：既避免了沙袋（base 太保守），又鼓励野心（stretch/moonshot）
缺点：奖金核算复杂，需要清晰的绩效管理系统

### 4.3 中层夹心（Middle-Out）

- BU 总经理层做核心（他们既懂业务又懂战略）
- 向上顶回 CEO：这个目标不可能
- 向下压回部门：这个是硬承诺

这是最贴近现实的模式。中层是预算成败的关键。

## 5. 预算游戏（Budget Games）与反制

部门在预算中的博弈策略，CFO 必须能识别：

### 5.1 常见游戏

| 游戏名 | 表现 | 反制 |
|---|---|---|
| 沙袋 | 明显低报能力 | 拉历史数据 + 同行基准挑战 |
| 冲花 | 12 月故意花光预算怕明年被砍 | 改成滚动预算或不砍未用预算 |
| 打包捆绑 | 把小项目打包成大预算好过审 | 要求单项目 Decision Package |
| 烟雾弹 | 提出一堆假需求，等被砍时留下真的 | 强制排序，只能砍不能加 |
| 甩锅假设 | "汇率不利 / 疫情反复 / 政策变化" | 假设写清楚 + 定期回顾谁的假设错得离谱 |
| 兄弟联盟 | 部门间协议一起不砍 | CFO 直接与业务对话，不通过部门总监 |
| 制造既成事实 | 预算未批就先招人先花钱 | 严格预算审批流程 + HR 联动 |

### 5.2 系统性反制

- **公开对账**：把各部门预算放在同一张表上，让部门互相看到（激发内部竞争）
- **多期数据支撑**：不接受"我觉得"，只接受"过去 3 年数据显示"
- **绩效挂钩**：预算准确度（±5% 内）也纳入个人考核
- **匿名反馈**：允许基层员工匿名指出"我们部门在沙袋"
- **审计追溯**：年终对严重偏离预算的做根因审计，找出是编制阶段的问题还是执行阶段的问题

```markdown
|---|---|---|---|---|---|---|---|
| 营收 | 12.0亿 | 10.5亿 | -1.5亿 | -12.5% | 销售部沙袋（历史准确率 90%） | 目标下压至 11.5，激励挂钩 | 11.5亿 |

```

## 7. 收敛失败的应对

如果三轮谈判后仍无法收敛（差异 >20%）：

1. **升级至董事会**：让董事会决定是"下调战略目标"还是"更换部门总监"
2. **临时预算**：先按部门数字执行 3 个月，Q2 根据实际再评估
3. **战略重议**：极端情况下，说明战略本身脱离实际，需要重新做 SWOT

**记住**：预算不是数学题，是组织政治。数字上能算对，人心上算不对，一样失败。
