# 投资决策方法（NPV / IRR / 回收期 / 获利指数）

投资决策的核心问题：给定一个项目（新工厂 / 新产品线 / 收购 / IT 系统 / 出海）需要投 X 亿，未来产生 Y 亿现金流，值不值得做？strategy-cfo 用这套方法把"值不值得"翻译成可量化、可比较、可挑战的数字。

## 一、四大主流指标

### 1. 净现值（Net Present Value, NPV）

$$
NPV = \sum_{t=0}^{n} \frac{CF_t}{(1+r)^t}
$$

- CF_t = t 期的净现金流（含初始投资 CF_0，通常为负）
- r = 折现率（一般用 WACC；风险大的项目用更高的项目特定折现率）
- n = 项目寿命期

**决策规则**：
- NPV > 0 → 项目创造股东价值，可做
- NPV < 0 → 项目摧毁股东价值，原则上不做
- NPV = 0 → 恰好覆盖资本成本，无所谓

**优点**：直接给出"这个项目值多少钱"，符合股东价值最大化目标
**缺点**：对折现率极敏感、对终值假设敏感、"绝对金额"难以直接横向比较

### 2. 内部收益率（Internal Rate of Return, IRR）

IRR 是使 NPV = 0 的折现率：

$$
0 = \sum_{t=0}^{n} \frac{CF_t}{(1+IRR)^t}
$$

**决策规则**：
- IRR > 资本成本（WACC） → 可做
- IRR < WACC → 不做

**优点**：直观（一个百分比）、易与其他机会比较、老板容易理解
**缺点**：
1. 非常规现金流可能有多个 IRR（数学上多解）
2. 互斥项目下 NPV 与 IRR 可能给出相反答案
3. **再投资假设错误**：IRR 隐含假设"中间现金流按 IRR 再投资"，实际做不到
4. 对规模不敏感（1 亿投资 IRR 30% vs 10 亿投资 IRR 20%，前者 IRR 高但绝对创造价值少）

### 3. 回收期（Payback Period）

**简单回收期**：多少年后累计名义现金流覆盖初始投资
**折现回收期**：多少年后累计折现现金流覆盖初始投资（更严格）

**决策规则**：与公司政策阈值比较（如"必须 5 年内回本"）

**优点**：简单、直观、关注短期风险
**缺点**：
1. 忽略回收期之后的现金流（可能项目回收期 6 年，但后面 20 年都赚大钱）
2. 简单回收期忽略时间价值
3. 不告诉你项目"值多少"

**用法**：**作为辅助指标而非主指标**。适合流动性紧张的公司、高不确定性项目（越晚的现金流越不可信）。

### 4. 获利指数（Profitability Index, PI）

$$
PI = \frac{PV(future\ cash\ flows)}{Initial\ Investment}
$$

**决策规则**：
- PI > 1 → 每 1 元投入创造 > 1 元价值，可做
- PI < 1 → 不做

**优点**：**资本受限时用来排序项目**（每 1 元资本创造多少价值）
**缺点**：可能鼓励做小项目

**关键用法**：当预算有限、多个正 NPV 项目要挑选时，按 PI 从高到低排序 → 直到预算用完。这比按 NPV 排序更能最大化总价值。

## 二、四大指标的组合使用

单一指标都有缺陷，实操上必须同时看：

| 指标 | 主要用途 | 权重 |
|---|---|---|
| NPV | 判断绝对价值创造 | 主指标 |
| IRR | 与其他机会比较 | 主指标 |
| 折现回收期 | 判断风险与流动性 | 辅助指标 |
| PI | 资本受限时排序 | 特殊情境 |

**决策模板**：
- NPV > 0 且 IRR > WACC + 风险溢价 → 通过
- NPV > 0 但回收期太长 → 需检查融资 / 流动性可行性
- NPV > 0 但 IRR 只勉强 > WACC → 敏感性分析后决定
- NPV < 0 但战略必要（卡位、防御、期权） → 明确量化战略溢价，需董事会特批

## 三、互斥项目 vs 独立项目

### 独立项目（Independent Projects）
彼此不干扰，都做也可以。**决策规则**：所有 NPV > 0 的都做（除非预算受限）。

### 互斥项目（Mutually Exclusive Projects）
只能选一个（如"建大工厂 vs 建小工厂"、"A 城 vs B 城"、"自建 vs 收购"）。

**互斥情境下 NPV 与 IRR 冲突的处理**：

假设两个互斥项目：
- 项目 A：投资 100，未来 10 年每年现金流 25 → IRR 21.4%，NPV（r=10%） = ¥54
- 项目 B：投资 1000，未来 10 年每年现金流 200 → IRR 15.1%，NPV（r=10%） = ¥229

按 IRR：选 A（21.4% > 15.1%）
按 NPV：选 B（¥229 > ¥54）

**正确答案：选 B**，因为 NPV 直接对应股东价值创造。IRR 忽略了项目规模。

**处理规则**：互斥项目下 NPV 优先，IRR 仅作参考。

### 增量 IRR（Incremental IRR）
如果非要用 IRR 判断互斥，用"增量 IRR"：计算 (B 现金流 − A 现金流) 的 IRR，与 WACC 比较。若增量 IRR > WACC → 从 A 升级到 B 值得。

上例：增量投资 900，增量现金流每年 175，增量 IRR ≈ 14.1% > 10% WACC → 从 A 升级到 B 值得 → 选 B（与 NPV 一致）。

### 修正内部收益率（Modified IRR, MIRR）

MIRR 修正了 IRR 的"再投资假设"缺陷：假定中间现金流按融资成本（更保守）再投资，最终现金流按项目 IRR 处理。

$$
MIRR = \left(\frac{FV(positive\ CF\ at\ finance\ rate)}{|PV(negative\ CF\ at\ discount\ rate)|}\right)^{1/n} - 1
$$

**用法**：作为 IRR 的替代或补充，尤其是长期项目 / 中间现金流大的项目。

## 四、敏感性分析（Sensitivity Analysis）

变动单个变量，看 NPV / IRR 如何变化，找出**最敏感的变量**。

### 典型敏感性表（针对 NPV）

| 变量 | 基线 | -20% | -10% | 基线 NPV | +10% | +20% |
|---|---|---|---|---|---|---|
| 收入增长率 | 15% | -¥0.5 亿 | ¥0.3 亿 | ¥1.2 亿 | ¥2.1 亿 | ¥3.0 亿 |
| 毛利率 | 30% | -¥1.0 亿 | ¥0.1 亿 | ¥1.2 亿 | ¥2.3 亿 | ¥3.4 亿 |
| WACC | 10% | ¥2.2 亿 | ¥1.7 亿 | ¥1.2 亿 | ¥0.8 亿 | ¥0.4 亿 |
| 初始投资 | ¥5 亿 | ¥2.2 亿 | ¥1.7 亿 | ¥1.2 亿 | ¥0.7 亿 | ¥0.2 亿 |
| 永续增长率 | 3% | ¥0.8 亿 | ¥1.0 亿 | ¥1.2 亿 | ¥1.5 亿 | ¥1.9 亿 |

**输出重点**：
1. 哪个变量斜率最陡 → 最敏感 → 应重点验证
2. 哪些变量组合会让 NPV 转负 → **红线组合**，写入退出条件
3. 敏感性图（蜘蛛图 / 龙卷风图）方便直观呈现

### 中国式敏感性变量清单
除通用变量外，中国项目还必看：
- 政策变化（补贴退坡、行业准入、环保标准）
- 汇率（出海项目、外币债务、进口原材料）
- 大客户依赖（前 5 客户占比 > 50% 的都要单独测试）
- 融资成本（LPR 上行 50 bp）
- 地方政府关系（税收优惠、土地成本、能耗指标）

## 五、情景分析（Scenario Analysis）

多个变量同时变动，构造"乐观 / 中性 / 悲观"三个（或更多）情景。

### 情景构造原则

- **乐观情景**：不是"每个变量都拉最好"，而是"合乎逻辑的最好组合"（例如高增长 + 高毛利 + 低 WACC 可能矛盾）
- **中性情景**：管理层基线预期
- **悲观情景**：符合逻辑的最坏组合（但排除黑天鹅）

### 加权期望值

| 情景 | 概率 | NPV | 加权 NPV |
|---|---|---|---|
| 乐观 | 25% | ¥3.0 亿 | ¥0.75 亿 |
| 中性 | 50% | ¥1.2 亿 | ¥0.60 亿 |
| 悲观 | 25% | -¥0.5 亿 | -¥0.125 亿 |
| **期望值** | 100% | — | **¥1.225 亿** |

**决策规则**：
- 期望 NPV > 0 → 值得考虑
- 悲观情景 NPV 是否可承受？如果亏损超过公司承受阈值 → 否决或分阶段做
- 情景之间跨度 → 反映项目不确定性

## 六、蒙特卡洛模拟（Monte Carlo Simulation）

情景分析的进阶：不是只算 3 个情景，而是给每个关键变量一个概率分布（正态 / 均匀 / 三角），随机采样 10000 次，得到 NPV 的分布。

### 实施思路（Python 伪代码）

```python
import numpy as np

# 变量分布
revenue_growth = np.random.normal(0.15, 0.05, 10000)  # 均值 15%，标准差 5%
gross_margin = np.random.normal(0.30, 0.04, 10000)
wacc = np.random.normal(0.10, 0.015, 10000)
initial_investment = 5e8

npvs = []
for i in range(10000):
    # 构建 10 年现金流
    cf = [-initial_investment]
    revenue = 10e8
    for year in range(10):
        revenue *= (1 + revenue_growth[i])
        cf_t = revenue * gross_margin[i] * 0.7  # 简化：毛利 × 转化为现金流的比例
        cf.append(cf_t / (1 + wacc[i]) ** (year + 1))
    npvs.append(sum(cf))

# 输出关键分位数
print(f"P5 (最悲观 5%): {np.percentile(npvs, 5)}")
print(f"P50 (中位数): {np.percentile(npvs, 50)}")
print(f"P95 (最乐观 5%): {np.percentile(npvs, 95)}")
print(f"P(NPV > 0): {np.mean([n > 0 for n in npvs])}")
```

**输出重点**：
- NPV 分布直方图（视觉上直观）
- P(NPV > 0) 概率
- 关键分位数（P5 / P25 / P50 / P75 / P95）
- 敏感性排序（哪些变量对 NPV 方差贡献最大 —— tornado chart）

**工具**：Python (numpy) 免费灵活；Excel 上 @Risk / Crystal Ball 是商业工具；专业 IB 用 Matlab / R。

**中国实操限制**：很多项目变量的历史数据不足以做严格的分布拟合，需要专家经验主观赋概率 —— 这时候蒙特卡洛的价值更多是"暴露不确定性范围"而非"精确预测"。

## 七、实物期权（Real Options）

传统 NPV 假设"决策一次到位，之后被动执行"。实际项目管理者拥有**后续调整权**——推迟、扩大、缩小、放弃、切换。这些"未来选择权"本身有价值，被传统 NPV 忽略。

### 四类主要实物期权

**1. 延迟期权（Option to Defer）**
- 场景：土地开发、矿业投资、新药 III 期
- 价值来源：等待更多信息（价格、需求、政策）
- 案例：油田公司在油价低时不开采（NPV 甚至负），持有权利本身有价值

**2. 扩张期权（Option to Expand）**
- 场景：试点项目、新市场入场
- 价值来源：先小规模验证，成功再扩大
- 案例：一家零售连锁在新城市开 3 家店试水，如果 IRR > 20% 就扩到 30 家
- **NPV 陷阱**：只算 3 家的现金流会低估战略价值

**3. 放弃期权（Option to Abandon）**
- 场景：可赎回资产、有二手市场的设备
- 价值来源：项目失败时能收回残值
- 案例：设备可转卖、门店可转租，让失败的绝对损失有下限

**4. 切换期权（Option to Switch）**
- 场景：多用途工厂、灵活生产线
- 价值来源：市场变化时可切换产品 / 输入 / 输出
- 案例：柔性生产线可切换车型（丰田 vs 传统车厂对比）；SHEIN 的小单快反本质是切换期权

### 定价方法

- **Black-Scholes 类模型**：把项目看作看涨期权，标的 = 项目现值，行权价 = 后续投资，波动率 = 现金流不确定性
- **决策树 / 二项树**：把项目分阶段，每阶段有多种可能路径 + 决策节点，倒推计算
- **蒙特卡洛 + 决策规则**：模拟中加入"如果 X 则 Y"的决策逻辑

### 实操建议

- 不要指望精确量化实物期权（学术模型对中国项目适用性有限）
- **重点在于识别**：这个项目是否有可延迟 / 扩张 / 放弃 / 切换的选项？如果有，传统 NPV 是低估的
- **给决策者一个"NPV + 期权价值"的区间**：例如"传统 NPV ¥1.2 亿，加上扩张期权保守估值 ¥3-5 亿"
- 涉及大型资本开支时，**分阶段决策的价值往往被低估** —— 花 10% 建试点，看结果再决定另外 90%，远好于一次投 100%

## 八、常见中国式投资决策场景

### 场景 1：新建工厂决策
- 关键变量：产能利用率（决定收入）、单位变动成本、固定制造费用
- 特有考量：地方政府补贴 / 税收优惠（要把优惠期算清楚，别把 5 年优惠期收益当作永久）、能耗指标、环保成本
- 陷阱：产能爬坡曲线过于乐观、竞争对手同时扩产被忽略

### 场景 2：并购决策
- 关键变量：目标估值、协同效应（收入协同 vs 成本协同）、整合成本、对赌条款影响
- 特有考量：商誉减值风险、跨文化整合成本、原团队流失
- 陷阱：协同效应普遍高估 50%+，整合成本普遍低估 30%+

### 场景 3：出海决策
- 关键变量：进入方式（自建 / 合资 / 收购）、汇率、当地税制、政治风险
- 特有考量：本地化团队搭建时间、地缘政治风险溢价（美国 / 欧盟 / 印度 / 东南亚差异极大）
- 陷阱：把国内成功经验直接复制、低估合规成本（GDPR、CCPA、当地劳工法）

### 场景 4：大型 IT / 数字化转型
- 关键变量：实施时间、员工采纳率、生产力提升幅度
- 特有考量：ROI 往往难以直接量化（需引入"避免的成本"、"缩短的决策时间"等间接指标）
- 陷阱：软件成本只是冰山一角，实施 + 培训 + 维护 = 3-5 倍软件成本

### 场景 5：新品类研发（生物医药、硬科技）
- 关键变量：研发成功概率（分阶段）、上市后市占率、单价、专利期
- 方法：**风险调整 NPV（rNPV）** —— 每阶段乘以成功概率
- 例：新药 rNPV = P(临床 I) × P(临床 II | I) × P(III | II) × P(上市 | III) × 上市后 NPV − 各阶段投入
- 中国医药创新药典型概率：临床 I 60% / II 40% / III 60% / 获批 80% ⇒ 累计约 11.5%

## 九、给 strategy-cfo 的用法建议

- 主指标默认给 NPV + IRR + 折现回收期，PI 在资本受限时补充。
- 互斥项目下 **NPV 优先**，IRR 只做参考；单一大项目下 IRR 更方便沟通。
- 每个项目至少给 **敏感性 + 情景** 两层不确定性分析；重大项目 + 高不确定性 → 加蒙特卡洛。
- 有分阶段可能性的项目 → 提示实物期权价值，不要用一次性 NPV 判死刑。
- 中国项目的敏感性变量清单必包含 **政策、汇率、大客户、融资成本、地方政府关系**。
- 涉及并购 / IPO / 融资等交易决策 —— **必须给 IRR 但不能只给 IRR**，因为规模效应会失真。
- 决策提交时明确标注：**NPV 为负但推荐做的**必须显式量化"战略溢价"（卡位、防御、期权），并说明"何种条件下战略溢价不成立时会退出"。
