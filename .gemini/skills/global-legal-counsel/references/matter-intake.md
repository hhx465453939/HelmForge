# Matter Intake

先把用户的问题锁进一个“可代理、可决策、可交付”的法律事项里。

## Intake Priority

优先问最少但最值钱的问题，不要一次性抛长问卷。

第一层必问：

1. 你要我站在哪一边？
2. 你希望达成什么结果？
3. 这件事现在处于什么阶段？
4. 适用哪个国家/地区/法院/监管机关？

第二层按需补问：

- 对手是谁，和你是什么关系
- 已发生的关键事实时间线
- 已有证据、文件、聊天、合同、流水、裁定、函件
- 你最担心对方会怎么说
- 截止日期、开庭日、签约日、回复监管期限
- 哪些结果绝对不能接受

## Scenario-specific Intake

### 1. 争议与诉讼

- 我方身份：原告/被告/申请人/被申请人
- 案由或拟定案由
- 当前程序阶段
- 诉请 / 对方请求
- 有无保全、冻结、刑事线索、并行程序

### 2. 合规与监管

- 监管对象是谁
- 已收到什么通知/问询/处罚
- 是否涉及持续违法风险
- 是否需要立即止损或留痕

### 3. 合同与交易

- 交易目标
- 当前博弈焦点条款
- 商业底线和法律底线
- 是否有时间压力或牌照/审批条件

### 4. 诈骗追偿 / 冻结 / 资金流

- 受害或涉案资金路径
- 已报警 / 已冻结 / 已保全情况
- 对方账户、上下游账户、其他冻结线索
- 对方可能的“善意”“代收代付”“正常交易”说法

## Intake Output

完成 intake 后，应生成一个简版事项定义：

```markdown
## 事项定义
- 委托方：
- 目标：
- 法域：
- 阶段：
- 对手：
- 核心争点：
- 已有证据：
- 最大缺口：
- 最近截止日期：
```
