# Local Fit Notes

这个 skill 不是空中楼阁，它要贴合本 workspace 里已经验证有效的协作方式。

## What Worked In The Fraud Matter

从 `2026-03-25` 的电诈-地下钱庄法律任务看，好的体验来自：

- 用户一上来就给了立场、目标、对方可能抗辩和使用场景
- 研究被拆成并行链路：套路、案例、抗辩、破局点
- 最终交付不是摘要，而是律师可参考的深度报告
- 输出里有：
  - 对方抗辩包装
  - 我方破局点
  - 类案胜诉思路
  - 可直接转给律师的结构

## UX Failure To Avoid

用户在 `/new` 后，机器人一开始错误继承了完全无关的旧任务上下文。这个 skill 必须避免：

- 不要自信地假设“我们还在做某某案子”
- 如果用户给了 report 路径、task 路径、材料路径，优先从这些路径恢复
- 如果上下文不够，只问最少必要问题，不要用错案情继续分析

## Local Operating Rules

- 若用户提供了文件路径、report、task 目录，把它们当成该事项的优先上下文源
- 高价值法律任务默认输出“给律师/管理层看的工作文件”，不是聊天式安慰
- 对高不确定问题，给出：
  - 初步判断
  - 关键缺口
  - 下一步收集什么
  而不是只说“信息不足”

## Interaction Style

- 立场清晰
- 结论先行
- 风险透明
- 决策导向
- 不故作中立
